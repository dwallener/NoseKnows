# NoseKnows Notebook

This notebook preserves observations, checkpoints, and design rationale that are useful context but too detailed for the active TODO list.

## Current Scope

- NoseKnows currently models one controlled fragrance exposure per capture.
- The learning problem should stay constrained until repeated real captures exist.
- Synthetic accuracy is useful for pipeline validation, not proof of real-world fragrance accuracy.
- No-scent/clean-air samples are first-class data. Correct no-scent behavior is silence, not a forced fragrance label.

## Dataset Layer Checkpoints

- Dataset construction stays outside the Rust model runtime. `README-DAFT.md` is the boundary document.
- Current recipe checkpoint: `recipes/peak_single_note.toml` selects no-scent plus single-note captures with `include_label_counts = [0, 1]`.
- Current materializer checkpoint: `scripts/materialize_dataset.sh` builds `data/manifest/captures.csv`, selected view CSVs under `data/views/`, and a view manifest.
- Current result-ledger checkpoint: `scripts/eval_peak_stream.sh` writes `data/runs/peak_stream/results.csv`; `scripts/report_peak_stream_run.sh` joins results to the manifest and writes grouped summaries.

## Stream and Live Runtime Notes

- The first continuous-training stream is constrained to one fragrance exposure at a time with explicit no-scent gaps.
- `scripts/build_snn_stream_dataset.sh` stitches generated capture datasets into long labeled streams with balanced single-note/two-note/three-note scent buckets, an initial no-scent prelude, and random whole-capture no-scent gaps after each scent segment.
- `scripts/train_snn_stream.sh` trains the separate stream readout model; it does not replace the capture-level accordion model.
- The current stream scaffold walks one row at a time, builds rolling baseline-relative rate/delta spike features, expands those features through the seeded 64-motif accordion state, and trains a sparse per-row `64 -> 14` readout.
- The peak-pair rolling stream evaluator is currently the cleaner demo path while the spike-stream model remains noisy.
- The Grid8 live path represents state as `8 active sensors x 8 one-second lookback buckets`.
- Carryover after an injection segment is expected in rolling-window representations and should be measured rather than hidden.

## Scent Embedding Notes

- `scent_embedding_v1` is an explicit ontology-backed 1024-dimensional bridge to NoseLLM.
- It preserves current 14-label output, recent history, pairwise label coactivation, and model-specific features.
- It should later be compared with a learned projection over the same live output/history/features.
- See `EMBEDDING.md` for the current dimension map.

## Synthetic Data Checkpoints

- Numeric-matrix and designer-phase generators are pipeline test fixtures.
- Comprehensive SNN synthetic training data is generated by `scripts/build_snn_training_dataset.sh`.
- Current comprehensive SNN training dataset target: `data/training/snn_comprehensive`, with no_scent=100, single_note=56, two_note=364, three_note=1456.
- Initial accordion smoke test on `data/training/snn_comprehensive` loaded 1976 captures and showed the objective was not yet blend/no-scent balanced: fragrance any@3 was high, but no-scent false positives remained at 100%.

- SNN LIF readouts carry a learned/exported per-label bias so no-scent silence survives fixed-point training and visualization.

- SNN encoder/training/export code stays separate from the transformer/NoseLLM path while reusing the same CSV datasets.

### Display Rubric Checkpoints

- `cargo run --bin snn_selftest` is the end-to-end SNN regression harness.
- Product-facing rubrics:
  - No-scent should produce silent gated readout.
  - Single-note should produce a longer-lived dominant signal on the correct label.
  - Two-note should make both expected labels visible, while allowing one to be weaker if the other is solid.
  - Three-note should make at least two of the three expected labels visible.
  - Small transient spillover on one or two other labels is acceptable when it does not dominate strongly.
- Use `--rubric display-no-scent`, `--rubric display-single`, `--rubric display-two`, and `--rubric display-three` as separate wheel acceptance checks.
- Use `--rubric display` / `--rubric display-all` only as aggregate checks.
- Keep the default strict rubric as a diagnostic for top-1/spillover analysis.
- Use `scripts/snn_selftest_full.sh` to regenerate stored self-test probes and run all four rubrics.

### Historical SNN Metrics

- Display-rubric checkpoint: 92/100 overall, 50/50 no-scent silent, 42/50 single-note pass.
- Segregated exhaustive-probe checkpoint: no-scent 50/50, single 9/14, two 10/91, three 53/364.
- Blend diagnostic: two-note had 69/182 expected slots visible, 4 weak, 109 missing; three-note had 283/1092 visible, 11 weak, 798 missing.
- Dominant blend failure attractor: `Dry Woods`, especially replacing `Amber`, `Fruity`, `Green`, `Soft Amber`, `Woody Amber`, and `Water`.
- Largest dominant-label confusions: Fruity -> Woods, Green -> Floral, Water -> Floral, and several Amber/Woods-family labels -> Silent.
- Failure split after targeted tuning: raw_silent=3, gate_silent=3, wrong_dominant=9, spillover=6, no_scent_fp=0.

### SNN Tuning Notes

- Base-note gate policy helped materially for `Floral Amber`, `Amber`, `Woody Amber`, and `Dry Woods`.
- Green/Water still lose to Floral; the next step should be better label-side contrast or motif discrimination rather than blindly increasing inhibition.
- `Citrus` should remain untouched while it stays clean.
- Do not prune `Woods` weights for `Fruity -> Woods` until contribution diagnostics identify responsible motifs.
- Distinguish tuning classes:
  - gate-silent heavy labels may need slower/per-label readout integration.
  - raw-silent labels need stronger pattern/label mapping before the gate can help.
  - wrong-dominant labels need label-side contrast or better discriminating motifs.
  - spillover-dominant labels need inhibition/calibration, not more sensitivity.

## Accordion / Differentiation Layer Notes

- The differentiation layer sits between spike generation and fragrance-label generation.
- Current Layer 1: 64 seeded emergent-pattern LIF neurons with winner-take-few lateral inhibition.
- Current local circuit rule: adc4-linked motifs use intrinsic threshold adaptation to attenuate sustained sticky-sensor tails.
- Current encoder guardrail: clean-air jitter below the absolute range/delta floor should not create spike activity.
- Current Layer 2: sparse supervised fixed-point mapping from emergent-pattern activity to the 14 fragrance-wheel labels.
- Future direction: replace or refine seeded mini-patterns with an unsupervised/self-organizing update rule.

## Gain Bridge Notes

- PLAN-002 defines the manual gain/attenuation bridge.
- Zero-input invariant: without a focus label, the gain stage must be exactly inert.
- Pegged sensors are real-world conditions. The first implementation reports clipping/pegging rather than hiding it.
- First implementation uses attenuate-only hand-authored masks for Grid8.
